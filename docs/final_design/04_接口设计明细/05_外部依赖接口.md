# 外部依赖接口

## 1. 说明

本文件描述当前正式方案依赖的外部接口：

- IDaaS
- IAM
- MCP 服务

这些接口不是我们要开发的主模块，但 Agent 网关和 MCP 网关需要与它们完成对接。

## 2. IDaaS 接口

### 2.0 IDaaS 委托信任前置条件

注册 Agent 后，外部系统会调用 IDaaS 的委托授权接口，建立 `Agent 网关 client_id` 与 `业务 Agent agent_id` 的信任关系。当前运行时链路只依赖该信任关系，不负责创建它。

运行时约定字段：

| 字段 | 说明 |
| --- | --- |
| `client_id` | Agent 网关在 IDaaS 的唯一 OAuth 客户端 ID |
| `client_secret` | Agent 网关 OAuth 客户端密钥，仅用于 `/oauth2/token` 的 `client_secret_basic` 认证 |
| `agent_id` | 业务 Agent 标识，全系统共用同一个值 |

### 2.1 `GET /oauth2/authorize`

#### 用途

- base 登录
- 业务授权

#### 请求示例：base 登录

```http
GET /oauth2/authorize?response_type=code&client_id=gw_client_001&agent_id=agt_business_001&redirect_uri=https%3A%2F%2Fagent-gateway.huawei.com%2Fgw%2Fauth%2Fbase%2Fcallback&scope=base.profile&state=gwst_001
```

#### 请求示例：业务授权

```http
GET /oauth2/authorize?response_type=code&client_id=gw_client_001&agent_id=agt_business_001&redirect_uri=https%3A%2F%2Fagent-gateway.huawei.com%2Fgw%2Fauth%2Fconsent%2Fcallback&scope=erp%3Acontract%3Ar%20erp%3Ainvoice%3Ar&state=gwst_auth_001
```

说明：

- `client_id` 表示 Agent 网关这个 OAuth 客户端。
- `agent_id` 表示当前业务 Agent，取值与 Agent 网关注册表中的 `agent_id` 一致。
- IDaaS 根据注册阶段已建立的委托信任关系校验 `client_id + agent_id` 是否匹配。

### 2.2 `POST /oauth2/token`

#### 用途

网关用授权 `code` 换取 `Tc` 对应的短令牌 `access_token`。

#### 请求示例

```http
POST /oauth2/token
Authorization: Basic base64(client_id:client_secret)
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&code=code_001&redirect_uri=https%3A%2F%2Fagent-gateway.huawei.com%2Fgw%2Fauth%2Fconsent%2Fcallback
```

字段说明：

- `code`：IDaaS callback 带回的授权码。
- `client_id`：Agent 网关在 IDaaS 的 OAuth 客户端 ID，通过 `Authorization: Basic` 传递。
- `client_secret`：Agent 网关 OAuth 客户端密钥，通过 `Authorization: Basic` 传递，不放入请求体。
- `redirect_uri`：可选；如果 IDaaS 要求校验 callback 地址，则必须与 authorize 阶段一致。

#### 响应示例

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiJ9...<Tc>...",
  "expires_in": 3600,
  "token_type": "Bearer",
  "scope": "base"
}
```

说明：

- base 登录阶段，Agent 网关在 `/gw/auth/ticket/exchange` 中调用该接口。
- 该接口只返回 `Tc access_token`，不直接作为浏览器回跳结果。
- Agent 网关拿到 `Tc access_token` 后，再调用 IDaaS 用户信息接口。
- 业务授权阶段，该接口返回的 `Tc` 会继续用于向 IAM 申请 `TR`。

### 2.3 `GET /oauth2/userinfo`

#### 用途

Agent 网关使用 `Tc access_token` 向 IDaaS 查询当前登录用户信息。真实路径以 IDaaS 接口文档为准，本文用 `/oauth2/userinfo` 表示用户信息接口。

#### 请求示例

```http
GET /oauth2/userinfo
Authorization: Bearer <Tc access_token>
```

说明：获取用户信息只需要 `access_token`，不再传 `client_id`、`client_secret` 或业务 Agent 信息。

#### 响应示例

```json
{
  "user_id": "z01062668",
  "uuid": "uuid~1234556",
  "username": "张三"
}
```

## 3. IAM 接口

### 3.1 `POST /iam/projects/{proxy_project_id}/assume_agent_token`

#### 用途

Agent 网关代表指定 Agent 申请 `T1`。

#### 请求头示例

```http
POST /iam/projects/{proxy_project_id}/assume_agent_token
Authorization: Bearer <iam_authorization_token>
Content-Type: application/json
```

说明：

- `Authorization` 后续由 `IAM SDK` 获取
- 当前开发联调阶段，可先从配置文件读取
- 代码中应显式保留 `TODO`

#### 请求体示例

```json
{
  "data": {
    "type": "assume_agent_token",
    "attributes": {
      "delegator_account_name": "Agent_agt_business_001",
      "delegator_appid": "com.huawei.business.agent",
    }
  }
}
```

说明：

- `delegator_account_name`：业务 Agent AI 账号名称，当前规则为 `Agent_` + `agent_id`
- `delegator_appid`：业务 Agent `appid`
- 当前两个值由 Agent 网关从 Agent 管理面查询出的 `agent_id / appid` 推导或读取

#### 成功响应示例

```json
{
  "message": "OK",
  "code": "201",
  "enterprise": "11111111111111111111111111111111",
  "access_token": "eyJhbGciOiJSUzUxMiIsInR5cCI6IkpXVCJ9....",
  "expires_at": "2026-04-03T14:24:48+08:00",
  "expires_in": 86399,
  "token_id": "1234",
  "token_type": "IAM_AI_SERVICE",
  "expires_on": 1775197488000
}
```

#### `T1 access_token` Payload 示例

说明：

- 下面不是 IAM 接口响应体中的直接字段
- 下面是响应体 `access_token` 解码后的 JWT payload
- `T1` 表达 Agent 程序集成身份，以及 IAM 对该 Agent 身份的认可

```json
{
  "token_id": "1234",
  "iss": "iam",
  "sub": "Agent_agt_business_001",
  "nbf": 1775024688,
  "exp": 1775197488,
  "iat": 1775111088,
  "jti": "6bfbbe98a-1d73-4bc7-a745-6efe94e9e0f9",
  "name": "Agent_agt_business_001",
  "account_id": "873a94228c9ea2ab018ca06c42e4000e",
  "enterprise": "11111111111111111111111111111111",
  "account_type": "IAM_AI_SERVICE",
  "project": "com.huawei.business.agent",
  "access_domain": "middle-secret",
  "scope": "erp:contract:r erp:invoice:r",
  "aud": "agt_business_001",
  "agent": {
    "agent_id": "agt_business_001",
    "agent_name": "ERP助手",
    "agent_type": "server"
  },
  "proxy_id": "com.huawei.business.agent"
}
```

#### `T1 access_token` Payload 字段说明

| 名称 | 类型 | 描述 |
| --- | --- | --- |
| `token_id` | String | token 唯一 ID；如果 IAM 真实 payload 不携带该字段，可使用响应体 `token_id` |
| `iss` | String | 签发人 / 令牌发行者 |
| `sub` | String | 主题 / 令牌代表的主体 |
| `nbf` | Number | 生效时间，Unix 时间戳，单位秒 |
| `exp` | Number | 过期时间，Unix 时间戳，单位秒 |
| `iat` | Number | 签发时间，Unix 时间戳，单位秒 |
| `jti` | String | JWT 唯一标识符，通常用于防重放 |
| `name` | String | 程序集成账号名称 |
| `account_id` | String | 程序集成账号 ID，由 IAM 维护 |
| `enterprise` | String | 企业 ID |
| `account_type` | String | 账号类型，此接口只支持 `IAM_AI_SERVICE` |
| `project` | String | appid |
| `access_domain` | String | 高 / 中 / 低秘 |
| `agent` | Object | 当前 Agent 信息 |
| `agent.agent_id` | String | Agent 唯一 ID |
| `agent.agent_name` | String | Agent 中文名称 |
| `agent.agent_type` | String | Agent 类型，当前默认 `server` |
| `proxy_id` | String | 代理的 appid，用于记录审计日志 |

### 3.2 `POST /iam/auth/resource-token`

#### 用途

Agent 网关用 `Tc + T1` 申请最终 `TR`。

#### 请求头示例

```http
POST /iam/auth/resource-token
Authorization: Bearer <agent_token>
Content-Type: application/json
```

说明：

- 这里的 `Authorization` 就是前一步获取到的 `T1`

#### 请求体示例

```json
{
  "data": {
    "type": "resource_token",
    "attributes": {
      "user_token": "Tc"
    }
  }
}
```

#### 成功响应示例

```json
{
  "message": "OK",
  "code": "201",
  "enterprise": "11111111111111111111111111111111",
  "access_token": "eyJhbGciOiJSUzUxMiIsInR5cCI6IkpXVCJ9....",
  "expires_at": "2026-04-03T14:24:48+08:00",
  "expires_in": 86399,
  "token_id": "1234",
  "token_type": "IAM_AI_SERVICE",
  "refresh_token": "11111111113",
  "expires_on": 1775197488000
}
```

#### `TR access_token` Payload 示例

说明：

- 下面不是 IAM 接口响应体中的直接字段
- 下面是响应体 `access_token` 解码后的 JWT payload
- `TR` 表达 Agent 代表用户访问资源，以及用户授权给 Agent 的权限点上限边界
- `consented_scopes` 是三方约定好的正式 wire format 字段名，不改名，格式为权限点 code 字符串数组
- `scope` 是预留字段，不作为运行时鉴权依据

```json
{
  "token_id": "1234",
  "iss": "iam",
  "sub": "com.huawei.business.agent",
  "nbf": 1775024688,
  "exp": 1775197488,

  "jti": "6bfbbe98a-1d73-4bc7-a745-6efe94e9e0f9",
  "name": "svc_ai_business_agent",
  "account_id": "873a94228c9ea2ab018ca06c42e4000e",
  "enterprise": "11111111111111111111111111111111",
  "account_type": "IAM_AI_SERVICE",
  "project": "com.huawei.business.agent",
  "access_domain": "middle-secret",
  "agent": {
    "agent_id": "agt_business_001",
    "agent_name": "ERP助手",
    "agent_type": "server"
  },
  "agency_user": {
    "idp": "idaas",
    "idp_id": null,
    "oauth_client_app_id": null,
    "user": "{\"uid\":\"z01062668\",\"uuid\":\"uuid~1234556\",\"displayNameCn\":\"张建国\",\"tenantId\":\"11111111111111111111111111111111\"}",
    "oauth_client_id": null,
    "consented_scopes": [
      "erp:contract:r",
      "erp:invoice:r"
    ]
  }
}
```

#### `TR access_token` Payload 字段说明

| 名称 | 类型 | 描述 |
| --- | --- | --- |
| `token_id` | String | token 唯一 ID；如果 IAM 真实 payload 不携带该字段，可使用响应体 `token_id` |
| `iss` | String | 签发人 / 令牌发行者 |
| `sub` | String | 主题 / 令牌代表的主体 |
| `nbf` | Number | 生效时间，Unix 时间戳，单位秒 |
| `exp` | Number | 过期时间，Unix 时间戳，单位秒 |
| `iat` | Number | 签发时间，Unix 时间戳，单位秒 |
| `jti` | String | JWT 唯一标识符，通常用于防重放 |
| `name` | String | AI 应用程序集成账号名称 |
| `account_id` | String | AI 应用程序集成账号 ID |
| `enterprise` | String | 企业 ID |
| `account_type` | String | 账号类型，固定为 `IAM_AI_SERVICE` |
| `project` | String | AI 应用所属项目 ID / appid |
| `access_domain` | String | 高 / 中 / 低秘 |
| `scope` | String | 预留字段，不作为运行时鉴权依据 |
| `aud` | String | 业务 Agent ID，运行时用于校验 TR 所属 Agent |
| `agent` | Object | 当前 Agent 信息 |
| `agent.agent_id` | String | Agent 唯一 ID |
| `agent.agent_name` | String | Agent 中文名称 |
| `agent.agent_type` | String | Agent 类型，当前默认 `server` |
| `agency_user` | Object | 被代理的用户信息及授权信息 |
| `agency_user.idp` | String | 身份提供者 |
| `agency_user.idp_id` | String | 身份提供者 ID |
| `agency_user.user` | String | 用户信息 JSON 字符串，运行时用户 ID 读取其中的 `uid` |
| `agency_user.oauth_client_id` | String | OAuth 客户端标识符 |
| `agency_user.oauth_client_app_id` | String | OAuth 客户端 appid |
| `agency_user.consented_scopes` | String[] | 用户授权给 Agent 的权限点 code 数组 |

## 4. MCP 服务接口

### 4.1 `POST /internal/v1/tools/invoke`

#### 用途

MCP 网关在完成 `TR` 范围校验和 Agent 策略判断后，转发到具体 MCP 服务。

#### 请求示例

```json
{
  "tool_id": "mcp:contract-server/get_contract",
  "arguments": {
    "contract_no": "HT-2026-001"
  }
}
```

#### 响应示例

```json
{
  "tool_id": "mcp:contract-server/get_contract",
  "result": {
    "contractNo": "HT-2026-001",
    "summary": "该合同当前状态为审批中，签约主体为华为技术有限公司。"
  }
}
```

## 5. 当前对接要求

- IDaaS 令牌中的权限点字段名固定为 `consented_scopes`
- `Tc` 和 `TR` 中的权限点 code 数组都沿用 `consented_scopes`
- `TR.scope` 是预留字段，不作为运行时鉴权依据
- 运行时用户 ID 从 `TR.agency_user.user` JSON 字符串里的 `uid` 读取
- 运行时 Agent ID 从 `TR.aud` 读取
- IAM 不负责解释权限点业务语义，只负责基于 `Tc + T1` 生成 `TR`
- T1 请求头中的 `Authorization` 后续由 `IAM SDK` 获取，当前阶段可先走配置文件
- T1 请求参数中的 `delegator_account_name` 按 `Agent_` + `agent_id` 拼接，`delegator_appid` 使用业务 Agent `appid`
- MCP 服务不负责做最终授权裁决；最终裁决在 MCP 网关完成
