# 外部依赖接口

## 1. 说明

本文件描述当前正式方案依赖的外部接口：

- IDaaS
- IAM
- MCP 服务

这些接口不是我们要开发的主模块，但 Agent 网关和 MCP 网关需要与它们完成对接。

## 2. IDaaS 接口

### 2.1 `GET /oauth2/authorize`

#### 用途

- base 登录
- 业务授权

#### 请求示例：base 登录

```http
GET /oauth2/authorize?response_type=code&client_id=gw_client_001&redirect_uri=https%3A%2F%2Fagent-gateway.huawei.com%2Fgw%2Fauth%2Fbase%2Fcallback&scope=base&state=gwst_001
```

#### 请求示例：业务授权

```http
GET /oauth2/authorize?response_type=code&client_id=gw_client_001&redirect_uri=https%3A%2F%2Fagent-gateway.huawei.com%2Fgw%2Fauth%2Fconsent%2Fcallback&scope=erp%3Acontract%3Ar%2Cerp%3Ainvoice%3Ar&state=gwst_auth_001
```

### 2.2 `POST /oauth2/token`

#### 用途

网关用授权 `code` 换取 `Tc` 或 base 登录结果。

#### 请求示例

```json
{
  "grant_type": "authorization_code",
  "code": "code_001",
  "redirect_uri": "https://agent-gateway.huawei.com/gw/auth/consent/callback",
  "client_id": "gw_client_001",
  "client_secret": "******"
}
```

#### 响应示例

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiJ9...<Tc>...",
  "expires_in": 3600,
  "user_id": "z01062668",
  "username": "张三",
  "consented_scopes": [
    {
      "code": "erp:contract:r",
      "displayNameZh": "ERP 合同的可读权限"
    },
    {
      "code": "erp:invoice:r",
      "displayNameZh": "ERP 发票的可读权限"
    }
  ]
}
```

## 3. IAM 接口

### 3.1 `POST /iam/projects/{proxy_project_id}/assume_agent_token`

#### 用途

Agent 网关代表指定 Agent 申请 `T1`。

#### 请求头示例

```http
POST /iam/projects/{proxy_project_id}/assume_agent_token
Authorization: Bearer <iam_authorization_token>
Content-Type: application/json
```

说明：

- `Authorization` 后续由 `IAM SDK` 获取
- 当前开发联调阶段，可先从配置文件读取
- 代码中应显式保留 `TODO`

#### 请求体示例

```json
{
  "data": {
    "type": "assume_agent_token",
    "attributes": {
      "delegator_account_name": "svc_ai_business_agent",
      "delegator_appid": "com.huawei.business.agent",
      "agent_id": "agt_business_001"
    }
  }
}
```

说明：

- `delegator_account_name`：委托方账号名称
- `delegator_appid`：委托方 `appid`
- `agent_id`：被委托方 `agent_id`
- 这三个值后续由 Agent 网关从数据库查询；当前阶段可先从配置文件读取，并预留 `TODO`

#### 成功响应示例

```json
{
  "message": "OK",
  "code": "201",
  "enterprise": "11111111111111111111111111111111",
  "access_token": "eyJhbGciOiJSUzUxMiIsInR5cCI6IkpXVCJ9....",
  "expires_at": "2026-04-03T14:24:48+08:00",
  "expires_in": 86399,
  "token_id": "1234",
  "token_type": "IAM_AI_SERVICE",
  "expires_on": 1775197488000
}
```

#### `T1 access_token` Payload 示例

说明：

- 下面不是 IAM 接口响应体中的直接字段
- 下面是响应体 `access_token` 解码后的 JWT payload
- `T1` 表达 Agent 程序集成身份，以及 IAM 对该 Agent 身份的认可

```json
{
  "token_id": "1234",
  "iss": "iam",
  "sub": "com.huawei.business.agent",
  "nbf": 1775024688,
  "exp": 1775197488,
  "iat": 1775111088,
  "jti": "6bfbbe98a-1d73-4bc7-a745-6efe94e9e0f9",
  "name": "svc_ai_business_agent",
  "account_id": "873a94228c9ea2ab018ca06c42e4000e",
  "enterprise": "11111111111111111111111111111111",
  "account_type": "IAM_AI_SERVICE",
  "project": "com.huawei.business.agent",
  "access_domain": "middle-secret",
  "agent": {
    "agent_id": "agt_business_001",
    "agent_name": "ERP助手",
    "agent_type": "server"
  },
  "proxy_id": "com.huawei.business.agent"
}
```

#### `T1 access_token` Payload 字段说明

| 名称 | 类型 | 描述 |
| --- | --- | --- |
| `token_id` | String | token 唯一 ID；如果 IAM 真实 payload 不携带该字段，可使用响应体 `token_id` |
| `iss` | String | 签发人 / 令牌发行者 |
| `sub` | String | 主题 / 令牌代表的主体 |
| `nbf` | Number | 生效时间，Unix 时间戳，单位秒 |
| `exp` | Number | 过期时间，Unix 时间戳，单位秒 |
| `iat` | Number | 签发时间，Unix 时间戳，单位秒 |
| `jti` | String | JWT 唯一标识符，通常用于防重放 |
| `name` | String | 程序集成账号名称 |
| `account_id` | String | 程序集成账号 ID，由 IAM 维护 |
| `enterprise` | String | 企业 ID |
| `account_type` | String | 账号类型，此接口只支持 `IAM_AI_SERVICE` |
| `project` | String | appid |
| `access_domain` | String | 高 / 中 / 低秘 |
| `agent` | Object | 当前 Agent 信息 |
| `agent.agent_id` | String | Agent 唯一 ID |
| `agent.agent_name` | String | Agent 中文名称 |
| `agent.agent_type` | String | Agent 类型，当前默认 `server` |
| `proxy_id` | String | 代理的 appid，用于记录审计日志 |

### 3.2 `POST /iam/auth/resource-token`

#### 用途

Agent 网关用 `Tc + T1` 申请最终 `TR`。

#### 请求头示例

```http
POST /iam/auth/resource-token
Authorization: Bearer <agent_token>
Content-Type: application/json
```

说明：

- 这里的 `Authorization` 就是前一步获取到的 `T1`

#### 请求体示例

```json
{
  "data": {
    "type": "resource_token",
    "attributes": {
      "user_token": "Tc"
    }
  }
}
```

#### 成功响应示例

```json
{
  "message": "OK",
  "code": "201",
  "enterprise": "11111111111111111111111111111111",
  "access_token": "eyJhbGciOiJSUzUxMiIsInR5cCI6IkpXVCJ9....",
  "expires_at": "2026-04-03T14:24:48+08:00",
  "expires_in": 86399,
  "token_id": "1234",
  "token_type": "IAM_AI_SERVICE",
  "refresh_token": "11111111113",
  "expires_on": 1775197488000
}
```

#### `TR access_token` Payload 示例

说明：

- 下面不是 IAM 接口响应体中的直接字段
- 下面是响应体 `access_token` 解码后的 JWT payload
- `TR` 表达 Agent 代表用户访问资源，以及用户授权给 Agent 的权限点上限边界
- `consented_scopes` 是三方约定好的正式 wire format 字段名，不改名

```json
{

  "iss": "iam",
  "sub": "com.huawei.business.agent",
  "nbf": 1775024688,
  "exp": 1775197488,

  "jti": "6bfbbe98a-1d73-4bc7-a745-6efe94e9e0f9",

  "enterprise": "11111111111111111111111111111111",
  "account_type": "IAM_AI_SERVICE",
  "project": "com.huawei.business.agent",
  "access_domain": "middle-secret",
  "agent": {
    "agent_id": "agt_business_001",
    "agent_name": "ERP助手",
    "agent_type": "server"
  },
  "agency_user": {
    "idp": "idaas",
    "idp_id": "2134",
    "user_id": "uuid~1234556",
    "global_user_id": "23131212121",
    "ouath_client_id": "23123",
    "outh_client_app_id": "12312",
    "consented_scopes": [
      {
        "code": "erp:contract:r",
        "displayNameZh": "ERP 合同的可读权限"
      },
      {
        "code": "erp:invoice:r",
        "displayNameZh": "ERP 发票的可读权限"
      }
    ]
  }
}
```

#### `TR access_token` Payload 字段说明

| 名称 | 类型 | 描述 |
| --- | --- | --- |
| `token_id` | String | token 唯一 ID；如果 IAM 真实 payload 不携带该字段，可使用响应体 `token_id` |
| `iss` | String | 签发人 / 令牌发行者 |
| `sub` | String | 主题 / 令牌代表的主体 |
| `nbf` | Number | 生效时间，Unix 时间戳，单位秒 |
| `exp` | Number | 过期时间，Unix 时间戳，单位秒 |
| `iat` | Number | 签发时间，Unix 时间戳，单位秒 |
| `jti` | String | JWT 唯一标识符，通常用于防重放 |
| `name` | String | AI 应用程序集成账号名称 |
| `account_id` | String | AI 应用程序集成账号 ID |
| `enterprise` | String | 企业 ID |
| `account_type` | String | 账号类型，固定为 `IAM_AI_SERVICE` |
| `project` | String | AI 应用所属项目 ID / appid |
| `access_domain` | String | 高 / 中 / 低秘 |
| `agent` | Object | 当前 Agent 信息 |
| `agent.agent_id` | String | Agent 唯一 ID |
| `agent.agent_name` | String | Agent 中文名称 |
| `agent.agent_type` | String | Agent 类型，当前默认 `server` |
| `agency_user` | Object | 被代理的用户信息及授权信息 |
| `agency_user.idp` | String | 身份提供者 |
| `agency_user.idp_id` | String | 身份提供者 ID |
| `agency_user.user_id` | String | 用户 UUID |
| `agency_user.global_user_id` | String | 用户全局 ID / 用户 ID |
| `agency_user.ouath_client_id` | String | OAuth 客户端标识符，字段名按当前 IAM 返回约定保留 |
| `agency_user.outh_client_app_id` | String | OAuth 客户端 appid，字段名按当前 IAM 返回约定保留 |
| `agency_user.consented_scopes` | Object[] | 用户授权给 Agent 的权限点对象数组 |
| `agency_user.consented_scopes[].code` | String | 权限点 code |
| `agency_user.consented_scopes[].displayNameZh` | String | 权限点中文描述 |

## 4. MCP 服务接口

### 4.1 `POST /internal/v1/tools/invoke`

#### 用途

MCP 网关在完成 `TR` 范围校验和 Agent 策略判断后，转发到具体 MCP 服务。

#### 请求示例

```json
{
  "tool_id": "mcp:contract-server/get_contract",
  "arguments": {
    "contract_no": "HT-2026-001"
  }
}
```

#### 响应示例

```json
{
  "tool_id": "mcp:contract-server/get_contract",
  "result": {
    "contractNo": "HT-2026-001",
    "summary": "该合同当前状态为审批中，签约主体为华为技术有限公司。"
  }
}
```

## 5. 当前对接要求

- IDaaS 令牌中的权限点字段名固定为 `consented_scopes`
- `Tc` 和 `TR` 中的权限点对象数组都沿用 `consented_scopes`
- IAM 不负责解释权限点业务语义，只负责基于 `Tc + T1` 生成 `TR`
- T1 请求头中的 `Authorization` 后续由 `IAM SDK` 获取，当前阶段可先走配置文件
- T1 请求参数中的 `delegator_account_name`、`delegator_appid`、`agent_id` 后续由 Agent 网关从数据库查询，当前阶段可先走配置文件
- MCP 服务不负责做最终授权裁决；最终裁决在 MCP 网关完成
