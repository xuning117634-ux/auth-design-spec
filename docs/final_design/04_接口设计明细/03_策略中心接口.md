# 策略中心接口

## 1. 模块职责

策略中心负责统一维护：

- 权限点定义
- 权限点与工具绑定关系
- Agent 与权限点订阅映射
- Agent 策略

它不负责：

- 浏览器跳转
- 登录会话
- `Tc / T1 / TR` 申请
- 运行时最终访问裁决

## 2. 接口清单

| 方法 | 路径 | 调用方 | 用途 |
| --- | --- | --- | --- |
| `PUT` | `/internal/v1/permission-points/batch-upsert` | MCP 网关 | 批量新增/更新权限点定义 |
| `POST` | `/internal/v1/permission-points/query` | 管理台 / Agent 网关 | 查询权限点定义 |
| `POST` | `/internal/v1/permission-points/resolve-by-tools` | Agent 网关 | 从工具集合反查权限点 |
| `POST` | `/internal/v1/permission-points/resolve-by-codes` | MCP 网关 | 从权限点反查工具集合 |
| `PUT` | `/internal/v1/agent-permission-points/batch-upsert` | Agent 网关 | 批量新增、恢复或停用 Agent 订阅权限点映射 |
| `PUT` | `/internal/v1/agent-strategies/batch-upsert` | Agent 网关 | 批量新增/更新 Agent 策略 |
| `POST` | `/internal/v1/agent-strategies/query` | MCP 网关 | 查询某个 Agent 针对权限点的策略 |

## 3. `PUT /internal/v1/permission-points/batch-upsert`

### 3.1 请求示例

```json
{
  "source": "mcp_gateway",
  "items": [
    {
      "permissionPointCode": "erp:contract:r",
      "enterprise": "ent_001",
      "appId": "erp",
      "displayNameZh": "ERP 合同的可读权限",
      "description": "允许读取 ERP 合同数据",
      "boundTools": [
        {
          "toolId": "mcp:contract-server/get_contract",
          "displayNameZh": "查询合同详情"
        }
      ],
      "status": "ACTIVE"
    },
    {
      "permissionPointCode": "erp:invoice:r",
      "enterprise": "ent_001",
      "appId": "erp",
      "displayNameZh": "ERP 发票的可读权限",
      "description": "允许读取 ERP 发票数据",
      "boundTools": [
        {
          "toolId": "mcp:invoice-server/query_invoices",
          "displayNameZh": "查询发票列表"
        }
      ],
      "status": "ACTIVE"
    }
  ]
}
```

### 3.2 响应示例

```json
{
  "upsertedCount": 2,
  "items": [
    {
      "permissionPointCode": "erp:contract:r",
      "result": "UPSERTED"
    },
    {
      "permissionPointCode": "erp:invoice:r",
      "result": "UPSERTED"
    }
  ]
}
```

### 3.3 规则

- 当前版本不新增独立删除接口
- `permissionPointCode` 在全系统保持唯一
- 每个权限点 item 必须携带 `enterprise` 和 `appId`
- `enterprise` 表示企业归属和隔离边界
- `appId` 表示权限点归属应用
- 权限点的停用和恢复统一通过 `batch-upsert + status` 完成
- `status=ACTIVE` 表示新增、更新或恢复
- `status=INACTIVE` 表示停用
- 停用后的权限点仍保留在存储中，但不再参与运行时映射
- `boundTools` 推荐传对象数组：`{toolId, displayNameZh}`
- 为兼容已有接入，`boundTools` 也接受字符串数组：`["mcp:contract-server/get_contract"]`
- 如果 `boundTools` 传字符串数组，策略中心会优先复用库中已有的工具中文名；若当前工具尚无中文名，则回退为 `toolId`

## 4. `POST /internal/v1/permission-points/query`

### 4.1 请求示例

按企业和应用查询：

```json
{
  "enterprise": "ent_001",
  "appId": "erp",
  "status": "ACTIVE",
  "permissionPointCodes": ["erp:contract:r"]
}
```

查询某个企业下全部应用的权限点：

```json
{
  "enterprise": "ent_001",
  "status": "ACTIVE"
}
```

### 4.2 响应示例

```json
{
  "permissionPoints": [
    {
      "permissionPointCode": "erp:contract:r",
      "enterprise": "ent_001",
      "appId": "erp",
      "displayNameZh": "ERP 合同的可读权限",
      "description": "允许读取 ERP 合同数据",
      "boundTools": [
        {
          "toolId": "mcp:contract-server/get_contract",
          "displayNameZh": "查询合同详情"
        }
      ],
      "status": "ACTIVE"
    }
  ]
}
```

### 4.3 规则

- `enterprise` 必填，不支持无企业条件的全局查询；不传时返回参数错误
- `appId` 可选，不传表示查询该 `enterprise` 下全部应用的权限点
- `permissionPointCodes` 可选，用于按权限点 code 精确过滤
- `status` 可选，用于按 `ACTIVE / INACTIVE` 过滤
- 无匹配结果时返回 `200 + 空数组`
- 该接口用于管理面或注册面查询，不参与 MCP 网关运行时裁决

## 5. `POST /internal/v1/permission-points/resolve-by-tools`

### 5.1 请求示例

```json
{
  "requiredTools": [
    "mcp:contract-server/get_contract",
    "mcp:invoice-server/query_invoices"
  ]
}
```

### 5.2 响应示例

```json
{
  "requiredPermissionPointCodes": [
    "erp:contract:r",
    "erp:invoice:r"
  ],
  "permissionPoints": [
    {
      "code": "erp:contract:r",
      "displayNameZh": "ERP 合同的可读权限"
    },
    {
      "code": "erp:invoice:r",
      "displayNameZh": "ERP 发票的可读权限"
    }
  ]
}
```

### 5.3 规则

- 返回结果去重并稳定排序
- 只基于 `ACTIVE` 权限点映射解析
- 如果 `requiredTools` 中的工具没有对应 `ACTIVE` 权限点映射，返回 `200 + 空数组`
- 工具与权限点映射不带 `agent` 维度，因此请求体不包含 `agentId`

## 6. `POST /internal/v1/permission-points/resolve-by-codes`

### 6.1 请求示例

```json
{
  "permissionPointCodes": [
    "erp:contract:r",
    "erp:invoice:r"
  ]
}
```

### 6.2 响应示例

```json
{
  "permissionPointCodes": [
    "erp:contract:r",
    "erp:invoice:r"
  ],
  "permissionPoints": [
    {
      "code": "erp:contract:r",
      "displayNameZh": "ERP 合同的可读权限"
    },
    {
      "code": "erp:invoice:r",
      "displayNameZh": "ERP 发票的可读权限"
    }
  ],
  "tools": [
    {
      "toolId": "mcp:contract-server/get_contract",
      "displayNameZh": "查询合同详情"
    },
    {
      "toolId": "mcp:invoice-server/query_invoices",
      "displayNameZh": "查询发票列表"
    }
  ]
}
```

### 6.3 规则

- 只返回 `ACTIVE` 权限点及其工具映射
- 如果请求中的权限点不存在、已停用，或没有对应运行时结果，返回 `200 + 空数组`
- 工具与权限点映射不带 `agent` 维度，因此请求体不包含 `agentId`

## 7. `PUT /internal/v1/agent-permission-points/batch-upsert`

### 7.1 请求示例

```json
{
  "agentId": "agt_business_001",
  "enterprise": "ent_001",
  "source": "agent-gateway",
  "items": [
    {
      "permissionPointCode": "erp:contract:r",
      "status": "ACTIVE"
    },
    {
      "permissionPointCode": "erp:invoice:r",
      "status": "INACTIVE"
    }
  ]
}
```

### 7.2 响应示例

```json
{
  "agentId": "agt_business_001",
  "enterprise": "ent_001",
  "upsertedCount": 2,
  "items": [
    {
      "permissionPointCode": "erp:contract:r",
      "result": "UPSERTED"
    },
    {
      "permissionPointCode": "erp:invoice:r",
      "result": "UPSERTED"
    }
  ]
}
```

### 7.3 规则

- `agentId` 必填
- `enterprise` 必填
- `permissionPointCode` 必须已存在于策略中心原始权限点表
- 请求只表达 `agentId + permissionPointCode` 的映射关系
- `status` 表示 Agent 与权限点订阅关系的状态，不表示权限点自身状态
- `status=ACTIVE` 表示订阅关系生效
- `status=INACTIVE` 表示取消或停用该订阅关系
- 当前该映射只作为注册面和管理面存储数据
- 当前不改变 `resolve-by-tools` 不带 `agentId` 的运行时口径
- 当前不参与 MCP 网关运行时裁决

## 8. `PUT /internal/v1/agent-strategies/batch-upsert`

### 8.1 请求示例

```json
{
  "agentId": "agt_business_001",
  "items": [
    {
      "strategyId": "stg_contract_permit_demo_user",
      "permissionPointCode": "erp:contract:r",
      "conditions": {
        "field": "subject.user_id",
        "operator": "in",
        "values": ["z01062668"]
      },
      "effect": "PERMIT",
      "status": "ACTIVE"
    },
    {
      "strategyId": "stg_invoice_deny_blocked_user",
      "permissionPointCode": "erp:invoice:r",
      "conditions": {
        "field": "subject.user_id",
        "operator": "in",
        "values": ["blocked_demo_user"]
      },
      "effect": "DENY",
      "status": "ACTIVE"
    }
  ]
}
```

### 8.2 响应示例

```json
{
  "agentId": "agt_business_001",
  "upsertedCount": 2,
  "items": [
    {
      "strategyId": "stg_contract_permit_demo_user",
      "result": "UPSERTED"
    },
    {
      "strategyId": "stg_invoice_deny_blocked_user",
      "result": "UPSERTED"
    }
  ]
}
```

### 8.3 规则

- 当前版本不新增独立删除接口
- 策略的停用和恢复统一通过 `batch-upsert + status` 完成
- `status=ACTIVE` 表示新增、更新或恢复
- `status=INACTIVE` 表示停用
- 停用后的策略仍保留在存储中，但 `agent-strategies/query` 不再返回

## 9. `POST /internal/v1/agent-strategies/query`

### 9.1 请求示例

```json
{
  "agentId": "agt_business_001",
  "permissionPointCodes": [
    "erp:contract:r",
    "erp:invoice:r"
  ]
}
```

### 9.2 响应示例

```json
{
  "agentId": "agt_business_001",
  "permissionPointCodes": [
    "erp:contract:r",
    "erp:invoice:r"
  ],
  "strategies": [
    {
      "strategyId": "stg_contract_permit_demo_user",
      "agentId": "agt_business_001",
      "permissionPointCode": "erp:contract:r",
      "conditions": {
        "field": "subject.user_id",
        "operator": "in",
        "values": ["z01062668"]
      },
      "effect": "PERMIT",
      "status": "ACTIVE"
    },
    {
      "strategyId": "stg_invoice_deny_blocked_user",
      "agentId": "agt_business_001",
      "permissionPointCode": "erp:invoice:r",
      "conditions": {
        "field": "subject.user_id",
        "operator": "in",
        "values": ["blocked_demo_user"]
      },
      "effect": "DENY",
      "status": "ACTIVE"
    }
  ]
}
```

### 9.3 规则

- 只返回 `ACTIVE` 策略
- 只按 `agentId + permissionPointCodes` 查询
- 如果请求中的权限点不存在、已停用，或当前 Agent 没有命中策略，返回 `200 + 空数组`

## 10. 当前版本约束

- 一个策略只绑定一个权限点
- `permissionPointCode` 全系统唯一
- 权限点必须携带 `enterprise` 和 `appId`
- 权限点查询接口中 `enterprise` 必填，`appId` 可选
- Agent 与权限点订阅映射由 `Agent 网关` 上报策略中心
- Agent 与权限点订阅映射当前只做存储，不参与运行时裁决
- 当前条件字段只支持 `subject.user_id`
- 当前运算符只支持 `equals` / `in`
- 权限点和策略都采用 `batch-upsert + status` 的状态删除模型
- `resolve-by-tools` 和 `resolve-by-codes` 都不带 `agentId`
- `resolve-by-tools` 和 `resolve-by-codes` 只处理 `ACTIVE` 权限点
- `agent-strategies/query` 仍然带 `agentId`，因为策略本身是 Agent 维度的
- 查询类接口在“无对应结果”时返回空数组，不抛业务异常
- 策略中心只负责存储和查询，不执行最终放行/拒绝裁决
