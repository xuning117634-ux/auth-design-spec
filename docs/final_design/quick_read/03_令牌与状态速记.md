# 03_令牌与状态速记

快速阅读摘要。**正式定义以** [03_令牌设计.md](../03_令牌设计.md) **、** [04_接口设计.md](../04_接口设计.md) **和** [05_策略中心设计.md](../05_策略中心设计.md) **为准**。

## 三类令牌职责

| 令牌 | 作用 | 谁负责获取 |
| --- | --- | --- |
| `Tc` | 表达用户身份与用户授权给 Agent 的权限点范围 | Agent 网关通过 `IDaaS /oauth2/token` 获取 |
| `T1` | 表达 Agent 身份与 IAM 对 Agent 的认可 | Agent 网关向 IAM 申请 |
| `TR` | 表达 Agent 代表用户访问资源的最终关系 | Agent 网关用 `Tc + T1` 向 IAM 申请 |

## 令牌里的权限点怎么表示

`Tc` 和 `TR` 都带：

```json
"authorizedPermissionPoints": [
  {
    "code": "erp:report:r",
    "displayNameZh": "ERP 报表的可读权限"
  }
]
```

## 字段归属速记

| 信息 | 当前归属 |
| --- | --- |
| 用户是谁 | `Tc.user`；进入 `TR` 后主要看 `agency_user.user_id / global_user_id` |
| 用户授权了哪些权限点 | `Tc.authorizedPermissionPoints`；进入 `TR` 后主要看 `agency_user.authorizedPermissionPoints` |
| Agent 是谁 | `T1.agent`；进入 `TR` 后继续看 `agent` |
| Agent 的接入身份是什么 | `agent_registry.agent_id + app_id + agent_service_account` |
| 最终当前用户可调用哪些工具 | `MCP 网关` 运行时基于“当前工具所需权限点 + Agent 策略 + TR 对应工具集合”计算 |

## 网关侧状态

| 对象 | 形态 | 作用 |
| --- | --- | --- |
| `gw_session` | `gw_session_id -> user_id, username, created_at` | 网关侧识别当前用户 |
| `gw_auth_context` | `(gw_session_id + agent_id) -> Tc, T1, TR, authorizedPermissionPoints, expires_at` | 记录当前用户在当前 Agent 下的授权上下文 |
| `pending_base_login` | `gw_state -> agent_id, return_url, outer_state` | 仅用于 base 登录回调恢复 |
| `pending_auth_transaction` | `request_id -> agent_id, required_tools, requiredPermissionPointCodes, missingPermissionPointCodes, return_url, gw_session_id, outer_state` | 对外恢复一次获取 `TR` 流程 |

## 业务 Agent 侧状态

| 对象 | 形态 | 作用 |
| --- | --- | --- |
| `site_session` | `site_session_id -> gw_session_token, user_id, username` | 业务网站自己的登录态 |
| `tr_cache` | `(site_session_id + agent_id) -> current_tr, covered_tools, coveredPermissionPointCodes, expires_at` | 覆盖式复用当前 `TR` |

## 当前固定口径

- `TR` 是用户授权给 Agent 的上限边界
- 策略是 Agent 所有者对不同用户功能开放范围的二次控制
- `MCP 网关` 运行时会先按当前工具反查所需权限点，再做策略判断，最后再校验该工具是否属于 `TR` 对应工具集合
- `request_id` 对业务 Agent 外显
- `gw_state` 只在网关内部出现
