# 03_令牌与状态速记

快速阅读摘要。**正式定义以** [03_令牌设计.md](../03_令牌设计.md) **和** [04_接口设计.md](../04_接口设计.md) **为准**。

## 三类令牌职责

| 令牌 | 作用 | 谁负责获取 |
| --- | --- | --- |
| `Tc` | 表达用户身份与用户授权范围 | Agent 网关通过 `IDaaS /oauth2/token` 获取 |
| `T1` | 表达 Agent 身份与 IAM 对 Agent 的认可 | Agent 网关向 IAM 申请 |
| `TR` | 表达 Agent 代表用户访问资源的最终关系 | Agent 网关用 `Tc + T1` 向 IAM 申请 |

## 字段归属速记

| 信息 | 当前归属 |
| --- | --- |
| 用户是谁 | `Tc.user`；进入 `TR` 后主要看 `agency_user.user_id / global_user_id` |
| 用户授权了哪些策略 code | `Tc.consented_scopes`；进入 `TR` 后主要看 `agency_user.consented_scopes` |
| Agent 是谁 | `T1.agent`；进入 `TR` 后继续看 `agent` |
| Agent 订阅了哪些工具 | `agent_registry.subscribed_tools` |
| 最终允许访问哪些工具 | `MCP 网关` 运行时基于 `agency_user.consented_scopes + agent_registry.subscribed_tools + 策略中心映射` 计算 |

## 网关侧状态

| 对象 | 形态 | 作用 |
| --- | --- | --- |
| `gw_session` | `gw_session_id -> user_id, username, created_at` | 网关侧识别当前用户 |
| `gw_auth_context` | `(gw_session_id + agent_id) -> Tc, T1, TR, consented_policy_codes, expires_at` | 记录当前用户在当前 Agent 下的授权上下文 |
| `pending_base_login` | `gw_state -> agent_id, return_url, outer_state` | 仅用于 base 登录回调恢复 |
| `pending_auth_transaction` | `request_id -> agent_id, required_tools, required_policy_codes, missing_policy_codes, return_url, gw_session_id, outer_state` | 对外恢复一次获取 `TR` 流程 |
| 内部 callback 关联 | `gw_state -> request_id` | 仅用于网关内部关联 OAuth callback |

## 业务 Agent 侧状态

| 对象 | 形态 | 作用 |
| --- | --- | --- |
| `site_session` | `site_session_id -> gw_session_token, user_id, username` | 业务网站自己的登录态 |
| `tr_cache` | `(site_session_id + agent_id) -> current_tr, covered_tools, covered_policy_codes, expires_at` | 覆盖式复用当前 `TR` |

## 覆盖式缓存怎么理解

业务 Agent 本地不再按“单次请求工具集合”单独建正式缓存，而是维护当前会话下的一条主缓存：

- `current_tr`
- `covered_tools`
- `covered_policy_codes`
- `expires_at`

判断规则：

1. 新请求进入时，比较 `required_tools` 是否被 `covered_tools` 覆盖
2. 覆盖且未过期，直接复用 `current_tr`
3. 不覆盖或已过期，再向网关重新申请

## 资源侧怎么得到最终可访问工具

`TR` 不直接承载 `granted_resources`。  
`MCP 网关` 在运行时做这 5 步：

1. 从 `TR.agency_user.consented_scopes` 读取用户授权的策略 code
2. 从 `agent_registry` 读取该 Agent 的 `subscribed_tools`
3. 通过策略中心做 `code -> tool` 映射
4. 将用户授权范围与 Agent 订阅能力做交集
5. 得到本次真正允许访问的工具集合

## 当前固定口径

- `Agent 网关` 负责 `tool -> code`
- `MCP 网关` 负责运行时 `code -> tool`
- `request_id` 对业务 Agent 外显
- `gw_state` 只在网关内部出现
