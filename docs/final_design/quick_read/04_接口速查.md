# 04_接口速查

快速阅读摘要。正式定义以 [04_接口设计.md](../04_接口设计.md) 和 [05_策略中心设计.md](../05_策略中心设计.md) 为准。

## 业务 Agent 后端长期需要理解的网关接口

| 调用方 | 被调用方 | 路径 | 输入 | 输出 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 业务 Agent | Agent 网关 | `POST /gw/auth/ticket/exchange` | `agent_id`、`ticketST` | 用户信息 | base 登录后创建 `site_session` |
| 业务 Agent | Agent 网关 | `POST /gw/token/resource-token` | `agent_id`、`required_tools`、`return_url`、`state`、可选 `subject_hint` | `redirect_url + request_id` | 创建授权事务 |
| 业务 Agent | Agent 网关 | `POST /gw/token/result/exchange` | `agent_id`、`request_id`、`token_result_ticket` | `TR` | 授权完成后换取 `TR` |

## 浏览器侧入口

| 调用方 | 被调用方 | 路径 | 作用 |
| --- | --- | --- | --- |
| 浏览器 | Agent 网关 | `GET /gw/auth/login` | 发起 base 登录 |
| 浏览器 | Agent 网关 | `GET /gw/auth/authorize` | 发起业务授权 |
| 浏览器 | 业务 Agent | `GET /agent?ticketST=...` | base 登录完成回跳 |
| 浏览器 | 业务 Agent | `GET /agent?token_result_ticket=...` | 授权完成回跳 |

## 网关内部依赖

| 调用方 | 被调用方 | 路径 | 用途 |
| --- | --- | --- | --- |
| Agent 网关 | IDaaS | `POST /oauth2/token` | 用 `code + client_id + client_secret + 可选 redirect_uri` 换 `Tc access_token` |
| Agent 网关 | IDaaS | `GET /oauth2/userinfo` | 用 `Tc access_token` 获取用户信息 |
| Agent 网关 | IAM | `POST /iam/projects/{proxy_project_id}/assume_agent_token` | 申请 `T1` |
| Agent 网关 | IAM | `POST /iam/auth/resource-token` | 用 `Tc + T1` 申请 `TR` |
| MCP 网关 | 策略中心 | `PUT /internal/v1/permission-points/batch-upsert` | 批量新增、停用或恢复权限点 |
| 管理台 / Agent 网关 | 策略中心 | `POST /internal/v1/permission-points/query` | 按 `enterprise` 查询权限点，`appId` 可选 |
| Agent 网关 | 策略中心 | `PUT /internal/v1/agent-permission-points/batch-upsert` | 全量覆盖 Agent 订阅权限点快照 |
| Agent 网关 | 策略中心 | `PUT /internal/v1/agent-strategies/batch-upsert` | 批量新增、停用或恢复 Agent 策略 |
| Agent 网关 | 策略中心 | `POST /internal/v1/permission-points/resolve-by-tools` | 反查本次所需权限点 |

## MCP 网关运行时依赖

| 调用方 | 被调用方 | 路径 | 用途 |
| --- | --- | --- | --- |
| MCP 网关 | 策略中心 | `POST /internal/v1/permission-points/resolve-by-tools` | 按当前待调用工具反查所需权限点 |
| MCP 网关 | 策略中心 | `POST /internal/v1/agent-strategies/query` | 查询这些所需权限点对应的 Agent 策略 |
| MCP 网关 | 策略中心 | `POST /internal/v1/permission-points/resolve-by-codes` | 按 `TR` 中权限点反查工具集合 |

## 当前固定规则

1. 业务 Agent 本地先判断是否有可复用 `TR`。
2. 无可用 `TR` 时，Agent 网关只返回授权跳转，不直接复用历史 `TR`。
3. 授权完成后，业务 Agent 用 `token_result_ticket` 后端换取 `TR`。
4. MCP 网关先按当前待调用工具反查所需权限点。
5. MCP 网关校验当前工具所需权限点是否在 `TR` 中。
6. MCP 网关做 Agent 策略判断。
7. MCP 网关按 `TR` 中权限点反查工具集合并校验当前工具是否在内。
8. 权限点查询接口必须传 `enterprise`；`appId` 可选，不传表示查询该企业下全部应用。
9. `permissionPointCode` 不能包含英文逗号 `,`。
10. Agent 订阅权限点快照使用全量覆盖语义，取消订阅时提交剩余完整权限点集合。
11. Agent 订阅权限点快照当前只做存储，不参与 MCP 网关运行时裁决。
