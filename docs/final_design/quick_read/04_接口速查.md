# 04_接口速查

快速阅读摘要。**正式定义以** [04_接口设计.md](../04_接口设计.md) **和** [05_策略中心设计.md](../05_策略中心设计.md) **为准**。

## 业务 Agent 长期只需要理解的接口

| 调用方 | 被调用方 | 路径 | 输入 | 输出 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 业务 Agent | Agent 网关 | `POST /gw/token/resource-token` | `gw_session_token`、`agent_id`、`required_tools`、`return_url`、`state` | `TR` 或 `redirect_url + request_id` | 获取 `TR` 的唯一核心运行时接口 |

## 浏览器侧入口

| 调用方 | 被调用方 | 路径 | 作用 |
| --- | --- | --- | --- |
| 浏览器 | Agent 网关 | `GET /gw/auth/login` | 发起 base 登录 |
| 浏览器 | Agent 网关 | `GET /gw/auth/authorize` | 发起业务授权 |

## 网关内部依赖

| 调用方 | 被调用方 | 路径 | 用途 |
| --- | --- | --- | --- |
| Agent 网关 | IDaaS | `POST /oauth2/token` | 用 code 换 base 登录结果或 `Tc` |
| Agent 网关 | IAM | `POST /iam/projects/{proxy_project_id}/assume_agent_token` | 申请 `T1` |
| Agent 网关 | IAM | `POST /iam/auth/resource-token` | 用 `Tc + T1` 申请 `TR` |
| Agent 网关 | 策略中心 | `POST /internal/v1/permission-points/resolve-by-tools` | 反查本次所需权限点 |

## MCP 网关运行时依赖

| 调用方 | 被调用方 | 路径 | 用途 |
| --- | --- | --- | --- |
| MCP 网关 | 策略中心 | `POST /internal/v1/permission-points/resolve-by-codes` | 根据权限点反查候选工具 |
| MCP 网关 | 策略中心 | `POST /internal/v1/agent-strategies/query` | 查询当前 Agent 的权限点策略 |

## 当前固定规则

1. 先做 `TR.authorizedPermissionPoints` 范围校验
2. 再做 Agent 策略判断
3. 两者都通过，当前工具才允许调用
