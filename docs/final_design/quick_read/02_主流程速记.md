# 02_主流程速记

快速阅读摘要。**正式定义以** [02_引入Agent网关版方案.md](../02_引入Agent网关版方案.md) **和** [04_接口设计.md](../04_接口设计.md) **为准**。

## 流程 A：base 登录

目标：

- 用户先拿到业务 Agent 的站点登录态
- 业务 Agent 不自己做 OAuth callback
- 网关完成 base 登录后，把 `gw_session_token` 带回业务 Agent

```mermaid
sequenceDiagram
    participant U as 用户浏览器
    participant BA as 业务 Agent
    participant GW as Agent 网关
    participant ID as IDaaS

    U->>BA: GET /agent
    BA->>BA: 检查 site_session_id
    alt 无站点登录态
        BA-->>U: 302 /gw/auth/login?agent_id=...&return_url=...&state=...
        U->>GW: GET /gw/auth/login
        GW->>GW: 写入 pending_base_login(gw_state)
        GW-->>U: 302 到 IDaaS /authorize?scope=base...
        U->>ID: GET /authorize?scope=base...
        ID-->>U: 302 /gw/auth/base/callback?code=...&state=gw_state_xxx
        U->>GW: GET /gw/auth/base/callback
        GW->>ID: POST /oauth2/token
        ID-->>GW: 返回 base 登录结果
        GW->>GW: 创建 gw_session + gw_session_token
        GW-->>U: 302 回业务 Agent 原页面（带 gw_session_token）
        U->>BA: GET /agent?gw_session_token=...
        BA->>BA: 创建 site_session
        BA-->>U: 302 /agent
    else 已有站点登录态
        BA-->>U: 直接返回页面
    end
```

业务 Agent 在这段里只需要记住：

1. 没登录就跳 `/gw/auth/login`
2. 网关回原页面时会带 `gw_session_token`
3. BFF 用 `gw_session_token` 建本地 `site_session`

## 流程 B：资源请求 / 获取 `TR`

目标：

- 业务 Agent 只上传 `required_tools`
- 网关决定是直接给 `TR`，还是先让用户授权
- 授权完成后，业务 Agent 重放原请求，再拿最终 `TR`

```mermaid
sequenceDiagram
    participant U as 用户浏览器
    participant BA as 业务 Agent
    participant GW as Agent 网关
    participant PC as 策略中心
    participant ID as IDaaS
    participant IAM as IAM
    participant MG as MCP 网关

    U->>BA: POST /chat/send
    BA->>BA: 判断 required_tools
    BA->>BA: 检查本地 TR 覆盖情况

    alt 本地 current_tr 有效且覆盖 required_tools
        BA->>MG: 带 TR 调 MCP
        MG-->>BA: 返回结果
        BA-->>U: 返回答案
    else 需要向网关申请
        BA->>GW: POST /gw/token/resource-token
        GW->>PC: resolve-by-tools(required_tools)
        PC-->>GW: required_policy_codes
        alt 网关已有覆盖这些 code 的有效 TR
            GW-->>BA: 直接返回 TR
            BA->>MG: 带 TR 调 MCP
            MG-->>BA: 返回结果
            BA-->>U: 返回答案
        else 需要用户授权
            GW-->>BA: 返回 redirect_url（其中带 request_id）
            BA-->>U: 前端跳转 redirect_url
            U->>GW: GET /gw/auth/authorize?request_id=...
            GW->>GW: 由 request_id 生成内部 gw_state
            GW-->>U: 302 到 IDaaS /authorize?scope=<required_policy_codes>
            U->>ID: GET /authorize?...
            ID-->>U: 302 /gw/auth/consent/callback?code=...&state=gw_state_xxx
            U->>GW: GET /gw/auth/consent/callback
            GW->>ID: POST /oauth2/token
            ID-->>GW: 返回 Tc
            GW->>IAM: 申请 T1
            IAM-->>GW: 返回 T1
            GW->>IAM: 用 Tc + T1 申请 TR
            IAM-->>GW: 返回 TR
            GW->>GW: 更新 gw_auth_context
            GW-->>U: 302 回业务 Agent 原页面
            U->>BA: 恢复原请求并再次调用 /gw/token/resource-token
            GW-->>BA: 返回 TR
            BA->>MG: 带 TR 调 MCP
            MG-->>BA: 返回结果
            BA-->>U: 返回答案
        end
    end
```

## 关键状态怎么理解

| 对象 | 谁看得到 | 作用 |
| --- | --- | --- |
| `gw_session_token` | 业务 Agent 可见 | 业务 Agent 调网关时的会话引用 |
| `request_id` | 业务 Agent 可见 | 一次“获取 `TR`”流程的外部流程号；通常体现在 `redirect_url` 中 |
| `gw_state` | 仅网关内部 | OAuth/browser callback 事务号，用于关联 IDaaS 回调 |
| `TR` | 业务 Agent、MCP 网关 | 最终资源访问令牌 |

## 业务 Agent 最小心智

- 只判断：
  - 本次要哪些 `required_tools`
  - 本地 `current_tr` 能不能覆盖
- 只处理两种结果：
  - 网关直接返回 `TR`
  - 网关返回 `redirect_url`
- 不自己处理：
  - OAuth callback
  - `Tc / T1 / TR` 编排
  - `policy_code`
