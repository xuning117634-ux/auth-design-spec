# Agent 安全令牌设计（OAuth2/JWT）

## 1. 说明

本文只描述当前版本的三类令牌：

- `Tc`：用户令牌
- `T1`：Agent 令牌
- `TR`：资源令牌

本文统一按 OAuth2/JWT 口径描述：

- 先给出接口请求体和响应体示例
- 再给出 `access_token` 解码后的 JWT Payload 示例
- 最后说明关键字段含义

本文暂不展开 `refresh_token` 接口设计。

## 2. 公共约定

### 2.1 标准 Claim

| Claim | 含义 |
|---|---|
| `iss` | 签发方 |
| `sub` | 令牌代表的主体 |
| `aud` | 目标接收方 |
| `nbf` | 生效时间 |
| `exp` | 过期时间 |
| `iat` | 签发时间 |
| `jti` | JWT 全局唯一 ID |

### 2.2 外层响应与 Payload 的关系

OAuth2 接口返回的通常是外层响应体，例如：

```json
{
  "code": "201",
  "message": "OK",
  "access_token": "eyJ...",
  "expires_at": "2026-04-08T11:00:00+08:00",
  "expires_in": 3600,
  "token_id": "rtk_001",
  "token_type": "Bearer"
}
```

其中：

- `access_token` 才是真正的 JWT
- JWT 解码后的内容才是本文里的 Payload 示例

## 3. Tc：用户令牌

`Tc` 由 `IDaaS` 签发，用来表达：

- 当前用户是谁
- 当前用户授权了哪些策略 code

### 3.1 接口说明

#### URI

```http
POST /oauth2/token
```

#### 请求体示例

```json
{
  "grant_type": "authorization_code",
  "code": "auth_code_001",
  "client_id": "agent_gateway_client",
  "redirect_uri": "https://agent-gateway.huawei.com/api/v1/auth/consent/callback"
}
```

#### 成功响应体示例

```json
{
  "code": "201",
  "message": "OK",
  "enterprise": "11111111111111111111111111111111",
  "access_token": "eyJhbGciOiJSUzUxMiIsInR5cCI6IkpXVCJ9.<Tc-Payload>.signature",
  "expires_at": "2026-04-08T11:00:00+08:00",
  "expires_in": 3600,
  "token_id": "utk_7c3f2e1d8b9a0654",
  "token_type": "Bearer"
}
```

#### 失败响应体示例

```json
{
  "timestamp": "2026-04-08T10:02:00+08:00",
  "id": "req_tc_001",
  "status": "401",
  "method": "POST",
  "path": "/oauth2/token",
  "title": "Authorization code invalid",
  "errors": [
    {
      "code": "INVALID_AUTHORIZATION_CODE",
      "message": "authorization_code is invalid or expired"
    }
  ]
}
```

### 3.2 请求参数

| 名称 | 类型 | 位置 | 描述 |
|---|---|---|---|
| `grant_type` | string | 请求体 | 固定为 `authorization_code` |
| `code` | string | 请求体 | IDaaS 授权回调返回的授权码 |
| `client_id` | string | 请求体 | Agent 网关在 IDaaS 注册的 OAuth2 客户端 ID |
| `redirect_uri` | string | 请求体 | 与授权时使用的回调地址保持一致 |

### 3.3 成功响应参数

| 名称 | 类型 | 描述 |
|---|---|---|
| `code` | string | HTTP 状态码 |
| `message` | string | 状态信息，成功时为 `OK` |
| `enterprise` | string | 企业 ID |
| `access_token` | string | `Tc` 对应的 JWT |
| `expires_at` | string | 过期时间 |
| `expires_in` | number | 多少秒后过期 |
| `token_id` | string | 外层 token ID |
| `token_type` | string | OAuth2 token 类型 |

### 3.4 Payload 示例

```json
{
  "iss": "idaas",
  "sub": "z01062668",
  "aud": "agent-gateway",
  "nbf": "2026-04-08T10:00:00+08:00",
  "exp": "2026-04-08T11:00:00+08:00",
  "iat": "2026-04-08T10:00:00+08:00",
  "jti": "utk_7c3f2e1d8b9a0654",
  "user": {
    "user_id": "z01062668",
    "uuid": "uuid-lj12541d2a461fdf",
    "username": "张建国",
    "email": "jianguo.zhang@huawei.com",
    "department": "财经财务管理部"
  },
  "consented_scopes": [
    "erp:report:read",
    "erp:report:category:list",
    "erp:invoice:read"
  ]
}
```

### 3.5 Payload 字段说明

| 字段 | 类型 | 含义 | 说明 |
|---|---|---|---|
| `iss` | string | 签发方 | 固定为 IDaaS |
| `sub` | string | 用户主体 | 当前取用户工号 |
| `aud` | string | 目标接收方 | 当前为 `agent-gateway` |
| `nbf` | string | 生效时间 | 使用格式化时间展示 |
| `exp` | string | 过期时间 | 使用格式化时间展示 |
| `iat` | string | 签发时间 | 使用格式化时间展示 |
| `jti` | string | JWT 唯一 ID | 用于防重放与审计 |
| `user` | object | 用户对象 | 承载真实用户信息 |
| `user.user_id` | string | 用户工号 | 业务侧可读用户标识 |
| `user.uuid` | string | 用户全局唯一标识 | 跨系统关联用户身份 |
| `user.username` | string | 用户姓名 | 审计与展示 |
| `user.email` | string | 用户邮箱 | 通知与展示 |
| `user.department` | string | 用户部门 | 资源侧可据此做隔离 |
| `consented_scopes` | string[] | 用户授权的策略 code | 例如 `erp:report:read` |

## 4. T1：Agent 令牌

`T1` 由 `IAM` 签发，用来表达：

- 当前 Agent 是谁
- IAM 是否认可这个 Agent 的应用身份

### 4.1 接口说明

#### URI

```http
POST /iam/projects/{proxy_project_id}/assume_agent_token
```

#### 请求体示例

```json
{
  "data": {
    "type": "assume_agent_token",
    "attributes": {
      "agent_service_account": "svc_ai_business_agent",
      "principal": "com.huawei.business.agent",
      "agent_id": "agt_business_001"
    }
  }
}
```

#### 成功响应体示例

```json
{
  "message": "OK",
  "code": "201",
  "enterprise": "11111111111111111111111111111111",
  "access_token": "eyJhbGciOiJSUzUxMiIsInR5cCI6IkpXVCJ9.<T1-Payload>.signature",
  "expires_at": "2026-04-09T10:00:00+08:00",
  "expires_in": 86400,
  "token_id": "atk_001",
  "token_type": "IAM_AI_SERVICE",
  "expires_on": 1775613600000
}
```

#### 失败响应体示例

```json
{
  "timestamp": "2026-04-08T10:05:00+08:00",
  "id": "req_t1_001",
  "status": "401",
  "method": "POST",
  "path": "/iam/projects/proxy_project_001/assume_agent_token",
  "title": "Authentication failed",
  "errors": [
    {
      "code": "INVALID_AGENT_ACCOUNT",
      "message": "agent_service_account is invalid"
    }
  ]
}
```

### 4.2 请求参数

| 名称 | 类型 | 位置 | 描述 |
|---|---|---|---|
| `agent_service_account` | string | 请求体 | AI 程序集成账号名称 |
| `principal` | string | 请求体 | Agent 应用 `appId` |
| `agent_id` | string | 请求体 | Agent 唯一 ID |

### 4.3 成功响应参数

| 名称 | 类型 | 描述 |
|---|---|---|
| `code` | string | HTTP 状态码 |
| `message` | string | 状态信息，成功时为 `OK` |
| `enterprise` | string | 企业 ID |
| `access_token` | string | `T1` 对应的 JWT |
| `expires_at` | string | 过期时间 |
| `expires_in` | number | 多少秒后过期 |
| `token_id` | string | 外层 token ID |
| `token_type` | string | 固定为 `IAM_AI_SERVICE` |
| `expires_on` | number | 过期毫秒时间戳 |

### 4.4 Payload 示例

```json
{
  "token_id": "atk_001",
  "iss": "iam",
  "sub": "com.huawei.business.agent",
  "nbf": "2026-04-08T10:00:00+08:00",
  "exp": "2026-04-09T10:00:00+08:00",
  "iat": "2026-04-08T10:00:00+08:00",
  "jti": "6bfbe98a-1d73-4bc7-a745-6efe94e9e0f9",
  "name": "svc_ai_business_agent",
  "account_id": "873a94228c9ea2ab018ca06c42e4000e",
  "enterprise": "11111111111111111111111111111111",
  "account_type": "IAM_AI_SERVICE",
  "project": "com.huawei.business.agent",
  "access_domain": "middle-secret",
  "agent": {
    "agent_id": "agt_business_001",
    "agent_name": "业务数据助手",
    "agent_type": "server"
  },
  "proxy_id": "com.huawei.business.agent"
}
```

### 4.5 Payload 字段说明

| 字段 | 类型 | 含义 | 说明 |
|---|---|---|---|
| `token_id` | string | token 唯一 ID | IAM payload 中保留 |
| `iss` | string | 签发方 | 固定为 IAM |
| `sub` | string | 主题 | 当前取 AI 应用的 `appId` |
| `nbf` | string | 生效时间 | 使用格式化时间展示 |
| `exp` | string | 过期时间 | 使用格式化时间展示 |
| `iat` | string | 签发时间 | 使用格式化时间展示 |
| `jti` | string | JWT 唯一 ID | 标准防重放字段 |
| `name` | string | 程序集成账号名称 | 对应 `agent_service_account` |
| `account_id` | string | 程序集成账号 ID | IAM 侧账号标识 |
| `enterprise` | string | 企业 ID | 企业级隔离与审计 |
| `account_type` | string | 账号类型 | 固定为 `IAM_AI_SERVICE` |
| `project` | string | 项目 / appId | AI 应用所属项目标识 |
| `access_domain` | string | 密级域 | 例如高/中/低秘 |
| `agent` | object | Agent 对象 | 承载 Agent 身份 |
| `proxy_id` | string | 代理 appId | 用于审计记录 |

### 4.6 说明

`T1` 不直接承载 `subscribed_tools`。  
`subscribed_tools` 保存在 `agent_registry`，用于后续和策略中心映射、用户授权范围一起计算最终可访问工具集合。

## 5. TR：资源令牌

`TR` 由 `IAM` 签发，用来表达：

- 当前 Agent 正在代表哪个用户
- 当前用户已经授权了哪些策略 code

### 5.1 接口说明

#### URI

```http
POST /iam/auth/resource-token
```

#### 请求体示例

```json
{
  "data": {
    "type": "resource_token",
    "attributes": {
      "user_token": "eyJhbGciOiJSUzUxMiIsInR5cCI6IkpXVCJ9.<Tc-Payload>.signature"
    }
  }
}
```

#### 成功响应体示例

```json
{
  "message": "OK",
  "code": "201",
  "enterprise": "11111111111111111111111111111111",
  "access_token": "eyJhbGciOiJSUzUxMiIsInR5cCI6IkpXVCJ9.<TR-Payload>.signature",
  "expires_at": "2026-04-08T11:05:00+08:00",
  "expires_in": 3600,
  "token_id": "rtk_001",
  "token_type": "IAM_AI_SERVICE",
  "expires_on": 1775617500000
}
```

> 说明：
> IAM 真实响应中可能还会包含 `refresh_token`，本文不展开。

#### 失败响应体示例

```json
{
  "timestamp": "2026-04-08T10:06:00+08:00",
  "id": "req_tr_001",
  "status": "401",
  "method": "POST",
  "path": "/iam/auth/resource-token",
  "title": "Authentication failed",
  "errors": [
    {
      "code": "INVALID_USER_TOKEN",
      "message": "user_token is invalid or expired"
    }
  ]
}
```

### 5.2 请求参数

| 名称 | 类型 | 位置 | 描述 |
|---|---|---|---|
| `Authorization` | string | 请求头 | `T1` 对应的 `access_token` |
| `user_token` | string | 请求体 | `Tc` 对应的 `access_token` |

### 5.3 成功响应参数

| 名称 | 类型 | 描述 |
|---|---|---|
| `code` | string | HTTP 状态码 |
| `message` | string | 状态信息，成功时为 `OK` |
| `enterprise` | string | 企业 ID |
| `access_token` | string | `TR` 对应的 JWT |
| `expires_at` | string | 过期时间 |
| `expires_in` | number | 多少秒后过期 |
| `token_id` | string | 外层 token ID |
| `token_type` | string | 固定为 `IAM_AI_SERVICE` |
| `expires_on` | number | 过期毫秒时间戳 |

### 5.5 Payload 字段说明

| 字段 | 类型 | 含义 | 说明 |

| `iss` | string | 签发方 | 固定为 IAM |

## 6. 三类令牌的职责

| 令牌 | 职责 |
|---|---|
| `Tc` | 表达用户身份与用户授权范围 |
| `T1` | 表达 Agent 身份与 IAM 对 Agent 的认可 |
| `TR` | 表达 Agent 代表用户访问资源的最终关系 |

## 7. 字段归属

| 信息 | 当前归属 |
|---|---|
| 用户是谁 | `Tc.user`；进入 `TR` 后主要看 `agency_user.user_id / global_user_id` |
| 用户授权了哪些策略 code | `Tc.consented_scopes`；进入 `TR` 后主要看 `agency_user.consented_scopes` |
| Agent 是谁 | `T1.agent`；进入 `TR` 后继续看 `agent` |
| Agent 订阅了哪些工具 | `agent_registry.subscribed_tools` |
| 最终允许访问哪些工具 | 运行时由 `agency_user.consented_scopes + agent_registry.subscribed_tools + 策略中心映射` 计算 |

## 8. 资源侧如何计算最终可访问工具

`TR` 不直接承载 `granted_resources`。  
资源侧按照下面的方式得到最终可访问工具集合：

1. 从 `TR.agency_user.consented_scopes` 读取用户授权的策略 code
2. 从 `agent_registry` 读取该 Agent 的 `subscribed_tools`
3. 通过 `Agent策略中心` 做 `code -> tool` 映射
4. 将用户授权范围与 Agent 订阅能力做交集
5. 得到本次真正允许访问的资源和工具集合
