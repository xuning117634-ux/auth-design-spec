## 2. 状态模型说明

- `security_requests` 保存单次安全编排的全过程状态
- `sessions` 保存可复用会话及当前 `TR`
- `tokens` 保存真实接口响应外壳与解码后的 payload
- `effective_resources` 保存在 `security_requests` / `sessions`，也可由运行时校验接口返回
- `effective_resources` 是运行时授权结果，不等于 `TR` 原始 payload

## 3. 典型交互场景

### 3.1 首次请求：需要登录

入口：

- `POST /api/v1/runtime/tr/acquire`

结果：

- 返回 `REDIRECT_REQUIRED + request_id + redirect_url`
- 创建 `security_request = req_sec_login_001`
- 当前不创建 `session`

对应数据对象：

- `security_requests.req_sec_login_001`

### 3.2 登录完成：进入授权

入口：

- `GET /api/v1/auth/login/callback`
- `POST /api/v1/runtime/tr/acquire`（带 `request_id`）

结果：

- 当前 `security_request` 写入 `gateway_user_context`
- 接入 Agent 再次获取 `TR` 时返回 `REDIRECT_REQUIRED + request_id + redirect_url`
- 网关开始根据 `subscribed_tools` 解析授权项

对应数据对象：

- `security_requests.req_sec_consent_001`

### 3.3 授权完成：进入 `READY`

入口：

- `GET /api/v1/auth/consent/callback`
- `POST /oauth2/token`
- `POST /iam/projects/{proxy_project_id}/assume_agent_token`
- `POST /iam/auth/resource-token`
- `POST /api/v1/runtime/tr/acquire`（带 `request_id`）

结果：

- 网关持有 `Tc / T1 / TR`
- 计算 `effective_resources`
- 创建 `session = ses_001`
- 返回 `READY + session_id + tr_token`

对应数据对象：

- `security_requests.req_sec_ready_001`
- `sessions.ses_001`
- `tokens.tc_access_token_001`
- `tokens.t1_access_token_001`
- `tokens.tr_access_token_001`

### 3.4 首次资源访问成功

入口：

- `POST /api/v1/mcp/invoke`
- `POST /internal/v1/tokens/tr/introspect`
- `POST /internal/v1/tools/invoke`

结果：

- `TR` 有效
- 工具命中 `effective_resources`
- MCP Server 返回业务数据

对应数据对象：

- `sessions.ses_001`
- `tokens.tr_access_token_001`
- `mcp_data.query_monthly_report`

### 3.5 `TR` 过期后刷新成功

入口：

- `POST /api/v1/runtime/sessions/{session_id}/tr/refresh`
- `POST /iam/auth/refresh-resource-token`

结果：

- 返回新的 `tr_token`
- 更新 `session.current_tr_ref`
- 更新 `session.current_refresh_token_ref`

对应数据对象：

- `sessions.ses_001`
- `tokens.tr_access_token_001`
- 新的 `tokens.tr_access_token_002`

### 3.6 `Tc` 失效后需要重新授权

入口：

- `POST /api/v1/runtime/sessions/{session_id}/tr/refresh`

结果：

- 网关发现 `tc_ref` 已失效
- 返回 `REDIRECT_REQUIRED + request_id + redirect_url`
- 不直接生成新的 `TR`

对应数据对象：

- `sessions.ses_003`
- `tokens.tc_access_token_expired_001`
- `tokens.tr_access_token_expired_001`
