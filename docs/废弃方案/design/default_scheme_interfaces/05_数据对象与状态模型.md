# 05. 数据对象与状态模型

## 1. 数据对象示例

### 1.1 `agent_registry`

```json
{
  "agent_registry": [
    {
      "agent_id": "agt_business_001",
      "agent_name": "业务数据助手",
      "appId": "com.huawei.business.agent",
      "version": "v2.1.0",
      "subscribed_tools": [
        "mcp:financial-report-server/query_monthly_report",
        "mcp:financial-report-server/list_report_categories",
        "mcp:invoice-server/query_invoices"
      ],
      "callback_base_url": "https://agent.local/chat"
    },
    {
      "agent_id": "agt_report_only_001",
      "agent_name": "报表只读助手",
      "appId": "com.huawei.report.agent",
      "version": "v1.3.2",
      "subscribed_tools": [
        "mcp:financial-report-server/query_monthly_report",
        "mcp:financial-report-server/list_report_categories"
      ]
    }

```

说明：

- `callback_base_url` 是可选字段，只用于回跳白名单或默认值
- 运行时主流程仍以 `client_return_url` 为准

### 1.6 `agent_runtime_bindings`

> 这是接入 Agent 后端/BFF的本地映射。它不是网关对象，用来把站点会话稳定映射到当前可复用的安全上下文。

```json
{
  "agent_runtime_bindings": [
    {
      "user_id": "z01062668",
      "agent_id": "agt_business_001",
      "site_session_id": "site_ses_001",
      "security_session_id": "sec_ses_001",
      "tr_token": "tr_access_token_001",
      "expires_at": "2026-04-02T10:15:00+08:00"
    }
  ]
}
```

说明：

- 安全复用键是：`user_id + agent_id + site_session_id`
- `conversation_id` 只属于业务对话维度，不作为安全上下文复用键
- 同一浏览器站点会话下的多个聊天页面可以复用同一条安全会话

## 2. 状态模型说明

- `security_requests` 保存单次安全编排的全过程状态
- `security_sessions` 保存可复用安全会话及当前 `TR`
- `agent_runtime_bindings` 保存接入 Agent 后端/BFF侧的本地映射
- `tokens` 保存真实接口响应外壳与解码后的 payload
- `effective_resources` 保存在 `security_requests / security_sessions`，也可由运行时校验接口返回
- `effective_resources` 是运行时授权结果，不等于 `TR` 原始 payload

## 3. 典型交互场景

### 3.1 首次请求：需要身份确认

入口：

- `POST /api/v1/runtime/tr/acquire`

结果：

- 返回 `REDIRECT_REQUIRED + request_id + redirect_url`
- 创建 `security_request = req_sec_login_001`
- 当前不创建安全会话
- 当前也不创建接入 Agent 后端/BFF的本地映射

对应数据对象：

- `security_requests.req_sec_login_001`

### 3.2 身份确认完成：进入授权

入口：

- `GET /api/v1/auth/login/callback`
- `POST /api/v1/runtime/tr/acquire`（带 `request_id`）

结果：

- 当前 `security_request` 写入 `gateway_user_context`
- 接入 Agent 再次获取 `TR` 时返回 `REDIRECT_REQUIRED + request_id + redirect_url`
- 网关开始根据 `subscribed_tools` 解析授权项

对应数据对象：

- `security_requests.req_sec_consent_001`

### 3.3 授权完成：进入 `READY`

入口：

- `GET /api/v1/auth/consent/callback`
- `POST /oauth2/token`
- `POST /iam/projects/{proxy_project_id}/assume_agent_token`
- `POST /iam/auth/resource-token`
- `POST /api/v1/runtime/tr/acquire`（带 `request_id`）

结果：

- 网关持有 `Tc / T1 / TR`
- 计算 `effective_resources`
- 创建 `security_session = sec_ses_001`
- 接入 Agent 后端/BFF 创建本地映射
- 返回 `READY + security_session_id + tr_token`

对应数据对象：

- `security_requests.req_sec_ready_001`
- `security_sessions.sec_ses_001`
- `agent_runtime_bindings[user_id + agent_id + site_session_id]`
- `tokens.tc_access_token_001`
- `tokens.t1_access_token_001`
- `tokens.tr_access_token_001`

### 3.4 首次资源访问成功

入口：

- `POST /api/v1/mcp/invoke`
- `POST /internal/v1/tokens/tr/introspect`
- `POST /internal/v1/tools/invoke`

结果：

- `TR` 有效
- 工具命中 `effective_resources`
- MCP Server 返回业务数据

对应数据对象：

- `security_sessions.sec_ses_001`
- `tokens.tr_access_token_001`
- `mcp_data.query_monthly_report`

### 3.5 同一站点会话下的后续请求复用

入口：

- 用户在同一浏览器站点会话下继续提问

结果：

- 接入 Agent 后端/BFF 先通过站点 cookie 找到 `site_session_id`
- 再用 `user_id + agent_id + site_session_id` 查本地映射
- 如果当前 `TR` 仍有效，直接复用已有：
  - `security_session_id`
  - `tr_token`
- 不需要重新授权

对应数据对象：

- `agent_runtime_bindings`
- `security_sessions.sec_ses_001`

### 3.6 `TR` 过期后刷新成功

入口：

- `POST /api/v1/runtime/security-sessions/{security_session_id}/tr/refresh`
- `POST /iam/auth/refresh-resource-token`

结果：

- 返回新的 `tr_token`
- 更新 `security_session.current_tr_ref`
- 更新 `security_session.current_refresh_token_ref`
- 更新接入 Agent 后端/BFF 本地映射中的：
  - `tr_token`
  - `expires_at`

对应数据对象：

- `security_sessions.sec_ses_001`
- `agent_runtime_bindings`
- 旧的 `tokens.tr_access_token_001`
- 新的 `tokens.tr_access_token_002`

### 3.7 授权失效后需要重新进入前置流程

入口：

- `POST /api/v1/runtime/security-sessions/{security_session_id}/tr/refresh`

结果：

- 网关发现当前授权不可复用
- 返回 `REDIRECT_REQUIRED + request_id + redirect_url`
- 不直接生成新的 `TR`
- 原安全会话保留，等待新的获取流程成功后再更新

对应数据对象：

- `security_sessions.sec_ses_001`
- 新的 `security_requests.req_sec_reauth_001`
