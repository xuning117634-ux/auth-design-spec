## 1. 模块边界总览

| 模块 | 主要职责 | 直接对接模块 | 不负责什么 |
|---|---|---|---|
| 接入Agent前端 | 页面展示、收集用户问题、按 `redirect_url` 跳转、回到原页面后继续当前流程 | 用户、接入Agent后端/BFF | 不直接调用 `IDaaS / IAM / MCP`，不感知 `request_id` 的内部语义，不持有 `security_session_id / tr_token` |
| 接入Agent后端/BFF | 调用 `Agent网关`、调用 `MCP`、维护站点会话到安全上下文的本地映射 | 接入Agent前端、Agent网关、MCP网关 | 不直接对接 `IDaaS`，不持有 `Tc / T1 / refresh_token` |
| Agent网关 | 登录启动、授权启动、回调处理、`Tc / T1 / TR` 的申请、校验与刷新、安全状态编排 | 接入Agent后端/BFF、IDaaS、IAM、Agent策略中心 | 不承接站点首页流量，不直接执行 MCP 工具 |
| IDaaS | 用户登录、用户授权、签发 `Tc` | Agent网关、用户浏览器 | 不管理 Agent 工具权限 |
| IAM | 签发 `T1`、签发 `TR`、刷新 `TR` | Agent网关 | 不管理页面跳转，不管理用户授权展示项 |
| Agent策略中心 | `tool -> code` 和 `code -> tool` 映射 | Agent网关 | 不直接签发 token |
| MCP网关 | 接受 `TR`、做运行时授权校验、转发工具调用 | 接入Agent后端/BFF、运行时校验服务、MCP Server | 不负责登录授权 |
| MCP Server | 执行具体工具调用并返回业务数据 | MCP网关 | 不直接处理 `Tc / T1` |

## 2. 核心接口总表

### 2.1 接入准备

| Phase | 调用方 | 被调用方 | 方法 | 路径 | 用途 | 关键输入 | 关键输出 | 状态影响 |
|---|---|---|---|---|---|---|---|---|
| 0 | 接入Agent后端/BFF 或平台配置 | Agent网关 | `POST` | `/api/v1/agents/register` | 注册 Agent 与订阅工具 | `agent_id`、`appId`、`subscribed_tools`、可选 `callback_base_url` | `registration_status` | 写入 `agent_registry` |

### 2.2 登录与授权编排

| Phase | 调用方 | 被调用方 | 方法 | 路径 | 用途 | 关键输入 | 关键输出 | 状态影响 |
|---|---|---|---|---|---|---|---|---|
| 1-3 | 接入Agent后端/BFF | Agent网关 | `POST` | `/api/v1/runtime/tr/acquire` | 获取或续取当前用户请求对应的 `TR` | 首次：`agent_id`、`user_request`、`client_return_url`；续取：`agent_id`、`request_id`、`client_return_url` | `READY + security_session_id + tr_token` 或 `REDIRECT_REQUIRED + request_id + redirect_url` | 创建或推进 `security_request` |
| 1 | 用户浏览器 | Agent网关 | `GET` | `/api/v1/auth/login/start` | 由网关统一启动登录 | `request_id`、`agent_id`、`client_return_url`、`state` | 302 到 `IDaaS` 登录页 | 写入 `login_context` |
| 1 | IDaaS | Agent网关 | `GET` | `/api/v1/auth/login/callback` | 登录成功回调 | `request_id`、`login_code`、`state` | 302 回跳 `client_return_url` | 写入网关侧用户身份上下文 |
| 2 | Agent网关 | Agent策略中心 | `POST` | `/internal/v1/policies/resolve-by-tools` | 按已订阅工具反查策略 code | `agent_id`、`subscribed_tools` | `policy_codes`、`policy_items` | 为授权阶段准备展示项 |
| 2 | 用户浏览器 | Agent网关 | `GET` | `/api/v1/auth/consent/start` | 由网关统一启动授权 | `request_id`、`state` | 302 到 `IDaaS` 授权页 | 写入 `consent_context` |
| 2 | IDaaS | Agent网关 | `GET` | `/api/v1/auth/consent/callback` | 授权成功回调 | `request_id`、`code`、`state` | 进入 `Tc` 获取阶段 | 写入授权码与授权结果 |
| 2 | Agent网关 | IDaaS | `POST` | `/oauth2/token` | 用授权码换取 `Tc` | `grant_type`、`code`、`client_id`、`redirect_uri` | `access_token` 作为 `Tc` | 写入 `tc_ref` |

### 2.3 `T1 / TR` 获取与首次安全会话建立

| Phase | 调用方 | 被调用方 | 方法 | 路径 | 用途 | 关键输入 | 关键输出 | 状态影响 |
|---|---|---|---|---|---|---|---|---|
| 3 | Agent网关 | IAM | `POST` | `https://apig.hisuat.huawei.com/iam/projects/{proxy_project_id}/assume_agent_token` | 获取 `T1` | `agent_service_account`、`principal`、`agent_id` | `access_token` 作为 `T1` | 写入 `t1_ref` 或临时缓存 |
| 3 | Agent网关 | IAM | `POST` | `https://apig.hisuat.huawei.com/iam/auth/resource-token` | 基于 `Tc + T1` 生成 `TR` | `Authorization: Bearer <t1_access_token>`、`user_token=<tc_access_token>` | `access_token` 作为 `TR`、`refresh_token` | 写入 `current_tr_ref`、`current_refresh_token_ref` |
| 3 | Agent网关 | Agent策略中心 | `POST` | `/internal/v1/tools/resolve-by-codes` | 按授权 code 正向解析工具集合 | `policy_codes` | `tool_mappings` | 计算 `effective_resources` |
| 3 | 接入Agent后端/BFF | Agent网关 | `POST` | `/api/v1/runtime/tr/acquire` | 在浏览器回跳后继续获取当前请求对应的 `TR` | `agent_id`、`request_id`、`client_return_url` | `READY + security_session_id + tr_token` 或 `REDIRECT_REQUIRED + request_id + redirect_url` | 首次创建安全会话或继续前置流程 |

### 2.4 资源访问与后续复用

| Phase | 调用方 | 被调用方 | 方法 | 路径 | 用途 | 关键输入 | 关键输出 | 状态影响 |
|---|---|---|---|---|---|---|---|---|
| 4 | 接入Agent后端/BFF | MCP网关 | `POST` | `/api/v1/mcp/invoke` | 带 `TR` 访问 MCP 工具 | `security_session_id`、`tr_token`、`server_id`、`tool_name`、`tool_input` | 工具执行结果 | 无直接状态推进 |
| 4 | MCP网关 | 运行时校验服务 | `POST` | `/internal/v1/tokens/tr/introspect` | 校验 `TR` 是否有效并返回授权范围 | `tr_token` | `active`、`expires_at`、`granted_resources` | 无直接状态推进 |
| 4 | MCP网关 | MCP Server | `POST` | `/internal/v1/tools/invoke` | 执行已授权工具调用 | `user`、`agent`、`tool_name`、`tool_input` | 业务数据 | 无直接状态推进 |
| 5 | 接入Agent后端/BFF | Agent网关 | `POST` | `/api/v1/runtime/security-sessions/{security_session_id}/tr/refresh` | 在 `TR` 过期或缺失时刷新 | `security_session_id`、`refresh_reason` | `READY + security_session_id + tr_token + expires_at` 或 `REDIRECT_REQUIRED + request_id + redirect_url` | 更新安全会话的当前 token 状态 |
| 5 | Agent网关 | IAM | `POST` | `https://apig.hisuat.huawei.com/iam/auth/refresh-resource-token` | 使用网关侧 `refresh_token` 刷新 `TR` | `refresh_token` | 新的 `access_token`、`refresh_token` | 更新 `current_tr_ref`、`current_refresh_token_ref` |

## 3. 状态推进接口

这一节只看“哪些接口会真正改变内部状态”。

| 接口 | 推进前 | 推进后 | 写入对象 | 说明 |
|---|---|---|---|---|
| `POST /api/v1/runtime/tr/acquire`（首次） | 无 | `LOGIN_REQUIRED` 或直接进入后续步骤 | `security_requests` | 创建一条新的安全请求主记录 |
| `GET /api/v1/auth/login/callback` | `LOGIN_REQUIRED` | 可进入 `CONSENT_REQUIRED` | `security_requests.gateway_user_context` | 登录回调写回用户身份上下文 |
| `POST /api/v1/runtime/tr/acquire`（续取） | 已登录未授权 | `CONSENT_REQUIRED` | `security_requests.consent_context` | 触发策略解析并准备授权项 |
| `GET /api/v1/auth/consent/callback` + `POST /oauth2/token` | `CONSENT_REQUIRED` | 可进入 `READY` 准备阶段 | `security_requests.tc_ref` | 回写授权码并获取 `Tc` |
| `POST /api/v1/runtime/tr/acquire`（READY 时） | 已有 `Tc` | `READY` | `security_sessions` | 获取 `T1 / TR`，计算 `effective_resources`，首次创建安全会话 |
| `POST /api/v1/runtime/security-sessions/{security_session_id}/tr/refresh` | `READY` 或 `TR_EXPIRED` | `READY / REDIRECT_REQUIRED` | `security_sessions` | 刷新成功则更新当前 `TR`，失败则返回新的前置流程入口 |

## 4. 评审时重点确认的接口边界

### 4.1 接入 Agent 侧只需要确认 3 个运行时接口

| 接口 | 接入 Agent 需要关心什么 |
|---|---|
| `POST /api/v1/runtime/tr/acquire` | 接入 Agent 后端/BFF 什么时候首次申请、什么时候带 `request_id` 续取、如何统一处理 `REDIRECT_REQUIRED` |
| `POST /api/v1/runtime/security-sessions/{security_session_id}/tr/refresh` | 接入 Agent 后端/BFF 在 `TR` 过期时怎么刷新 |
| `POST /api/v1/mcp/invoke` | 接入 Agent 后端/BFF 如何带 `security_session_id + tr_token` 调工具 |

### 4.2 Agent 网关侧要重点确认的内部能力链路

| 能力链路 | 关键接口 |
|---|---|
| 登录启动与回调 | `/api/v1/auth/login/start`、`/api/v1/auth/login/callback` |
| 默认方案授权项生成 | `/internal/v1/policies/resolve-by-tools` |
| `Tc` 获取 | `POST /oauth2/token` |
| `T1 / TR` 获取 | `assume_agent_token`、`resource-token` |
| `TR` 刷新 | `refresh-resource-token` |

### 4.3 资源访问侧重点确认的边界

| 边界 | 说明 |
|---|---|
| 只认 `TR` | MCP 不接受 `Tc / T1` |
| 运行时校验独立存在 | 当前通过 `/internal/v1/tokens/tr/introspect` 承担 |
| 授权范围是运行时结果 | `granted_resources` 不是 `TR` 原始 payload 字段 |

## 5. 一页版最小结论

如果评审时只想记住最关键的结论，可以压成下面这 5 句：

1. 接入 Agent 只对接 `Agent网关` 和 `MCP网关`，不直接对接 `IDaaS`
2. 登录、授权、`Tc / T1 / TR` 都由 `Agent网关` 统一编排
3. 接入 Agent 运行时只对外看 3 个接口：获取、刷新、使用
4. 后续复用靠 `security_session_id + tr_token`，并由接入 Agent 后端/BFF 基于 `user_id + agent_id + site_session_id` 做本地映射
5. `TR` 过期时只允许通过 `POST /api/v1/runtime/security-sessions/{security_session_id}/tr/refresh` 回网关刷新
